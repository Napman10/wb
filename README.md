1. Работа в сервисах обернута в транзакции для возможных добавлений новых функций с гарантией того, что при ошибке данные будут откачены, а при успешной работе - зафиксированы

2. /hire - нанятие сотрудика. Вписываются нужные поля из ТЗ, генерится id и вставляется в базу
пример запроса
curl --location 'http://0.0.0.0:1111/hire' \
--header 'Content-Type: application/json' \
--header 'Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ= \
--data-raw '{
    "fullname": "test",
    "gender": 1,
    "age": 20,
    "email": "test@test.test",
    "address": "address test"
}'

3. /fire - увольнение сотрудника. Выставляется конкретная дата удаления без физического уничтожения данных (soft delete)
пример запроса
curl --location --request DELETE 'http://0.0.0.0:1111/fire?id=fefc8d3c-540a-40b3-a77d-7750da600953 --header 'Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ='

4. /vacation_days - получение количества дней отпуска. Просто достается значение из соответствующего поля в базе, на данном этапе значение не модифицируется сервисом
пример запроса
curl --location 'http://0.0.0.0:1111/vacation_days?id=fefc8d3c-540a-40b3-a77d-7750da600953 --header 'Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ='

5. /search - поиск по полю ФИО или хотя бы его началу
пример запроса
curl --location 'http://0.0.0.0:1111/search?query=test' \
--header 'Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ='

6. На уровне middleware добавлена примитивная basic авторизация, на данном этапе на соответствие хардкод данным

7. Приложение поделено на три слоя - транспорт, сервис, данные

8. Для запуска инфраструктуры (в текущем случае только базы) применить с корня проекта
   ```docker-compose -f ./infrastructure/docker-compose.yaml up```

9. Для записи env-переменных вызвать с корня проекта ```/bin/zsh ./env.sh```
    Предлагаемые порты для БД и сервера специфические, так как привычные были заняты

10. Создание таблиц будет запущено автоматически
